# FinQA RAG Assistant

Система ответов на финансовые вопросы по документам. Решение строит ответ на основе найденных фрагментов, сочетая поиск, переранжирование и генерацию.

## Назначение

По коллекции финансовых документов находить фрагменты, достаточные для ответа на вопрос, и формировать ответ, опирающийся на найденные источники.

## Архитектура

```text
Финансовые документы
        │
        ▼
Разбиение на фрагменты
        │
        ▼
Векторное представление
        │
        ▼
Поиск top-K фрагментов
        │
        ▼
Переранжирование
        │
        ▼
Генерация ответа по найденным фрагментам
```

## Что реализовано

- загрузка и подготовка документов;
- разбиение документов на фрагменты;
- получение векторных представлений;
- поиск релевантных фрагментов;
- переранжирование найденных фрагментов;
- генерация ответа по найденному контексту;
- отдельный упрощённый сценарий сравнения в корне репозитория (рядом с `main.py`).

## Структура репозитория

- `src/data_loader.py` — загрузка и подготовка документов;
- `src/embedding_service.py` — получение векторных представлений;
- `src/retriever.py` — поиск релевантных фрагментов;
- `src/generator.py` — формирование ответа;
- `src/pipeline.py` — полный конвейер обработки;
- `main.py` — точка входа;
- `questions.csv`, `train_data.csv` — данные для экспериментов.

## Оценка качества

Основные метрики:

- `Recall@K` для этапа поиска;
- доля вопросов, для которых релевантный фрагмент найден в `top-K`;
- изменение качества после переранжирования;
- задержка по этапам;
- доля ответов, подтверждённых найденными источниками.

Численные значения приводятся по итогам сохранённых отчётов после прогона сценариев оценки в этом репозитории.

## Результаты

Табличные значения не приводятся: в репозитории нет зафиксированного автоматического отчёта с итоговыми числами после стандартного прогона. После запуска оценки их переносят из сформированных файлов отчёта.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python main.py
```

Упрощённый контур сравнения запускается вторым скриптом в корне репозитория (тот же уровень каталога, что и `main.py`); имя файла совпадает с единственным дополнительным `.py` в корне, кроме `main.py`.

## Ограничения

- качество генерации зависит от полноты найденных фрагментов;
- отдельно контролируют качество поиска и качество ответа;
- для воспроизводимого сравнения сохраняют результаты упрощённого и полного контура в отдельных отчётах.
