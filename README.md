# FinQA RAG Assistant

## Кратко
Система ответов на вопросы по финансовым документам, в которой ответ строится только на основе найденных фрагментов корпуса. Основной конвейер включает подготовку документов, поиск релевантных фрагментов, их переранжирование и генерацию ответа с опорой на найденный контекст.

## Задача
В вопросах по финансовым документам особенно важно снизить риск недостоверных ответов. Вместо генерации ответа «из памяти модели» используется схема, в которой система сначала находит релевантные фрагменты документов, затем упорядочивает их по полезности для ответа и только после этого формирует итоговый текст.

## Что улучшено
- документы приводятся к форме, удобной для поиска по фрагментам;
- поиск релевантных фрагментов отделён от генерации ответа;
- добавлено переранжирование найденных фрагментов перед ответом;
- снижён риск ответов, не подтверждённых источником.

## Архитектура
```text
[корпус финансовых документов]
              |
              v
[разбиение документов на фрагменты]
              |
              v
[построение представлений и индекс]
              |
              v
[поиск релевантных фрагментов]
              |
              v
[переранжирование]
              |
              v
[генерация ответа по найденному контексту]
```

## Структура репозитория
```text
src/           основная логика системы
baseline.py    базовый режим для сравнения
main.py        точка входа
questions.csv  примеры вопросов
train_data.csv данные корпуса
```

## Критерии качества
- качество поиска релевантных фрагментов;
- качество верхних позиций после переранжирования;
- доля ответов, подтверждённых найденным контекстом;
- задержка ответа.

## Ключевой вывод
Разделение системы на поиск, переранжирование и генерацию делает ответы более управляемыми и снижает риск недостоверных формулировок. Главный практический эффект даёт не только генерация, а качество найденного и правильно упорядоченного контекста.

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

## Ограничения
- качество ответа ограничено полнотой корпуса и качеством поиска;
- слишком мелкие или слишком крупные фрагменты могут ухудшать итоговый результат;
- при росте числа документов нужно отдельно контролировать задержку поиска и переранжирования.
